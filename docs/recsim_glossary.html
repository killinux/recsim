🎯1. 强化学习基础概念
MDPMarkov Decision Process
RL 的数学框架。每个时间步：Agent 观察状态 → 选择动作 → 获得奖励 → 转移到新状态。RecSim 中：用户状态=state，推荐 slate=action，用户反馈=reward。
POMDPPartially Observable MDP
Agent 看不到完整状态，只能看到部分观测。RecSim 的核心设计——LTS 中 Agent 看不到 satisfaction，IEx 中看不到用户兴趣细节。
Episode回合
一次完整的用户会话。从用户进入系统到离开（time_budget ≤ 0），期间 Agent 与环境反复交互。
Reward奖励
Agent 每步获得的即时反馈。不同环境定义不同：LTS=engagement，IE=watched_time，IEx=是否点击(0/1)。
Discount Factor (γ)折扣因子
衡量对未来奖励的重视程度。γ=0 只看当下（贪心），γ=0.99 近乎同等重视未来。推荐系统需要高 γ 做长期规划。
Exploration vs Exploitation探索 vs 利用
探索：尝试不确定的选项以获取信息。利用：选已知最好的选项。IEx 环境专门考察此权衡。
📋2. 推荐系统特有概念
Slate推荐列表
一次推荐给用户的一组文档，是 Agent 的"动作"。代码中为整数索引数组：action = np.array([0, 3, 7])，从候选集中选第 0、3、7 个文档。
Combinatorial Action Space组合动作空间
从 N 个候选选 K 个组成 slate，动作空间 = C(N,K)。如 N=100, K=5 有 7500 万种组合。SlateDecompQ 将复杂度从 O(C(N,K)) 降为 O(N)。
Dynamic Candidate Set动态候选集
候选内容池不断变化——新内容上线、旧内容下架。resample_documents=True 模拟此动态性。
Clickbait标题党
高即时吸引力但低实质价值的内容。LTS 环境中用 clickbait_score ∈ [0,1] 量化，越高越"标题党"。
CTR / pCTR点击率 / 预测点击率
Click-Through Rate。GreedyPCTRAgent 只按预测点击率排序推荐，不考虑长期影响，是短视基线。
🏗️3. RecSim 架构概念
Environment环境 · simulator/environment.py
仿真核心。编排每步交互：接收 slate → 模拟用户响应 → 更新状态 → 返回观测和奖励。核心类 SingleUserEnvironment。
User Model用户模型 · user.py
封装用户完整行为：状态管理 + 响应生成 + 状态转移 + 终止判断。核心类 AbstractUserModel。
Document Sampler文档采样器 · document.py
生成候选文档集 CandidateSet。每步可重新采样模拟动态内容池。
RecSimGymEnvGym 封装 · simulator/recsim_gym.py
将 RecSim 环境包装为 OpenAI Gym 接口（reset()、step()），可接入任何标准 RL 库。
Reward Aggregator奖励聚合函数
将多维用户响应（点击、时长、喜欢等）压缩为标量 reward。如 clicked_watchtime_reward、clicked_engagement_reward。
Gin Config配置系统
Google 超参数配置框架。@gin.configurable 标记可配置类，命令行即可调参：--gin_bindings="Class.param=value"
交互循环：reset() → Agent 出 slate → Environment.step(slate) → simulate_response → update_state → 返回 (obs, reward, done)
👤4. 用户建模概念
User State用户状态
用户某时刻的完整内部状态。含隐变量（兴趣/满意度，Agent 看不到）和可观测变量（人口统计等）。create_observation() 决定暴露哪些。
Partial Observability部分可观测性
环境Agent 能看到Agent 看不到
LTS空（什么都看不到）satisfaction, NPE
IE用户兴趣向量—
IEx用户类型 ID对各主题的具体偏好
Response用户响应
用户对推荐文档的反馈。IE: clicked/watch_time/liked/quality/cluster_id；LTS: clicked/engagement；IEx: clicked。
Time Budget时间预算
用户 session 剩余时间。每步消耗（看视频耗时），高质量推荐可补回一些。耗尽则 episode 结束。
Score Function评分函数
用户对文档的偏好分数，通常为兴趣向量与文档特征的点积：
score = user_interests · doc_features
🎲5. 选择模型概念
Multinomial Logit多项式 Logit · choice_model.pyIEx
对分数做 softmax 采样。score 越高选中概率越大，但低分文档仍有机会。
P(x) = exp(score(x)) / Σy exp(score(y))
Cascade Model级联模型IE
用户从上到下依次浏览 slate：看第 1 个 → 以概率 p₁ 点击(点了就停) → 没点 → 以 attention_prob 的概率继续看第 2 个 → …。位置越靠后被浏览到的概率越低，模拟位置偏差。
No-Click Mass不点击质量
用户"什么都不点"的倾向。值越高越容易划过整个 slate 不点。类比：用户看了推荐页觉得都不感兴趣，直接关 app。
🍫6. Long-Term Satisfaction 环境
核心困境：巧克力(clickbait)短期好吃但不健康，甘蓝(优质内容)短期没那么诱人但有长期价值。好的策略要像营养师一样搭配。
NPENet Positive Exposure · 净正面曝光
核心隐变量。衡量用户累计接收内容质量的"计分器"。推优质内容 NPE 上升，推 clickbait NPE 下降。
NPEt+1 = γ · NPEt − 2·(clickbait_score − 0.5) + ε,  ε ~ N(0, σ²)
γ·NPE_t 遗忘效应(γ=0.7)  |  clickbait>0.5 → NPE 下降  |  clickbait<0.5 → NPE 上升  |  ε 为情绪随机波动
Satisfaction满意度（隐变量）
从 NPE 推导，取值 (0,1)。NPE 高→满意度接近 1→用户活跃；NPE 低→满意度接近 0→推什么都没人看。
satisfaction = 1 / (1 + exp(−sensitivity · NPE))
Engagement参与度
Agent 唯一能观测到的信号。由文档吸引力和满意度共同决定：
μ = (clickbait · choc_mean + (1−clickbait) · kale_mean) × satisfaction
engagement = exp(N(μ, σ²))
choc_mean(5.0) > kale_mean(4.0)，clickbait 短期 engagement 更高但会降低 satisfaction。
Memory Discount (γ)记忆衰减因子
默认 0.7。控制用户遗忘速度。越高则过去体验影响越持久。
Sensitivity敏感度
满意度公式中的系数。越高则 NPE 微小变化就引起满意度剧烈波动。
🔄7. Interest Evolution 环境
核心特点：用户兴趣可观测且会随推荐内容动态漂移。好策略要利用已知兴趣做精准推荐，同时推高质量内容延长会话。
User Interests用户兴趣向量
20 维向量，每维表示对一个主题的偏好 ∈ [-1,1]。+1=非常感兴趣，-1=厌恶。此环境中 Agent 能看到此向量。
Interest Drift兴趣漂移
观看文档后，兴趣以一定概率向文档特征靠近或远离。越强的兴趣越不容易改变（α 随 |interests| 减小）。
α(x) = (−y_int / x_int) · |x| + y_int
Δinterests = α · mask · (doc_features − interests)
类比：铁杆球迷不会因看一个烹饪视频就不喜欢足球，但对烹饪中性的人可能因此产生兴趣。
Utility效用
用户从观看文档中获得的综合价值。高效用推荐延长会话（补回时间预算），低效用加速用户离开。
utility = user_quality_factor × expected_utility + doc_quality_factor × doc_quality
budget += α_update × watch_time × utility
Cluster ID聚类 ID
每个文档属于 20 个主题聚类之一，特征为 one-hot 编码。用于聚合分析各主题推荐效果。
Watch Time观看时长
min(time_budget, video_length)。是 reward（clicked_watchtime_reward）的直接来源。
🔍8. Interest Exploration 环境
核心特点：用户兴趣是隐变量且不随时间变化。本质是多臂赌博机问题，考察 Agent 能否通过探索发现用户真正偏好。
User Type用户类型
用户分为 N 种类型，每种对不同主题有不同偏好模式。Agent 只能看到类型 ID，看不到具体偏好值。
Production Quality制作质量 fD(d)
文档的固有质量，与用户无关。高制作质量的文档在所有用户中点击率都较高。
Affinity亲和度
用户对文档的最终偏好，两部分组成：
affinity = gU(u,d) + fD(d) = 用户个人偏好 + 文档制作质量
Popularity Bias流行度偏差
短视策略偏向推荐高制作质量的大众主题（对所有用户点击率都不低），忽略小众但用户真正感兴趣的主题。
Correlated Bandits相关臂赌博机
不同主题的 reward 是相关的（共享制作质量这个混淆因素）。需要比标准 UCB1 更精细的探索策略来去除混淆。
🤖9. Agent 与算法
Agent策略适用场景
Random随机选文档最弱基线
Greedy pCTR按预测点击率排序选 top-K短视基线，只看即时 CTR
TabularQ离散化状态空间的 Q-learning小规模问题
FullSlateQ整个 slate 作为一个动作，DQN组合爆炸，仅适用于小 slate
SlateDecompQ分解 slate Q 值为单文档 Q 值之和论文重点，O(N) 复杂度
UCB1置信上界，自动平衡探索/利用Bandit 问题(IEx)
ClusterBanditUCB1 在文档聚类上的应用聚类级别的探索
UCB1 公式
UCB(a) = 平均reward + √(2 · ln(总次数) / 该臂被拉次数)
被拉次数少 → 不确定性大 → 置信上界高 → 值得探索。
Hierarchical Agent Layers分层 Agent
类似 Keras layer 的可堆叠设计。特征工程层（TemporalAggregation、FixedLengthHistory）+ 算法层（TabularQ、DQN），可独立替换。
🔧10. 外部基准与工具
Atari街机游戏模拟器
RL 经典基准。输入=游戏画面(像素)，输出=手柄操作。DeepMind 2013 年 DQN 在此达到人类水平。代表：Breakout、Pong、Space Invaders。
MuJoCoMulti-Joint dynamics with Contact
物理仿真引擎，模拟机器人关节运动。Agent 控制力矩完成走路/跑步/抓取。代表：HalfCheetah、Humanoid、Ant。
为什么推荐系统不能用 Atari / MuJoCo
特性Atari / MuJoCo推荐系统
动作空间固定几个按键/力矩C(N,K) 组合爆炸
候选集不变每步有新内容上线/下架
用户状态完全可观测兴趣/满意度是隐变量
目标单一得分多目标(CTR+时长+满意度)
OpenAI Gym
RL 环境标准接口：reset()、step(action)、observation_space、action_space。RecSim 通过 RecSimGymEnv 适配。
Dopamine
Google 的 RL 研究框架，提供 DQN 等参考实现。RecSim 的 FullSlateQ 和 SlateDecompQ 基于此。
TensorFlow
Google 深度学习框架。RecSim 原始代码基于 TF 1.x，本项目已适配 TF 2.x（将 import gin.tf 改为 import gin）。
